# 12. 配置、异常处理与通用组件

## 启动配置

### 数据源

```yaml
spring:
  datasource:
    url: jdbc:mysql://${MYSQL_HOST:127.0.0.1}:${MYSQL_PORT:3306}/${MYSQL_DATABASE:review_platform}?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
    username: ${MYSQL_USERNAME:root}
    password: ${MYSQL_PASSWORD:}
```

环境变量优先，冒号后是默认值。密码默认空，只适合本地环境。

`serverTimezone=UTC` 会影响 JDBC 时间转换，需要与数据库、应用服务器时区保持一致。

### Redis

```yaml
spring:
  redis:
    host: ${REDIS_HOST:127.0.0.1}
    port: ${REDIS_PORT:6380}
    password: ${REDIS_PASSWORD:}
```

默认端口是 6380，不是 Redis 常见的 6379。启动前要核对本机实例。

### Lettuce 连接池

```yaml
lettuce:
  pool:
    max-active: 10
    max-idle: 10
    min-idle: 1
    time-between-eviction-runs: 10s
```

Stream 阻塞读取会长期占一个连接，因此真实部署要关注连接池耗尽，而不是只看 HTTP 线程数。

## MyBatis 配置

```yaml
mybatis-plus:
  type-aliases-package: com.aschen.redis.entity
  configuration:
    log-impl: org.apache.ibatis.logging.slf4j.Slf4jImpl
```

开发环境打印 SQL 便于学习，但生产环境 debug SQL 可能暴露数据、产生大量日志并降低性能。

## Jackson 配置

```yaml
spring:
  jackson:
    default-property-inclusion: non_null
```

Result 中为 null 的 `errorMsg/data/total` 不会出现在 JSON，减小响应体，但前端不能依赖 null 字段一定存在。

## 全局异常处理

```java
@ExceptionHandler(RuntimeException.class)
public Result handleRuntimeException(RuntimeException e) {
    log.error(e.toString(), e);
    return Result.fail("服务器异常");
}
```

优点：

- 不把堆栈返回前端。
- 所有 RuntimeException 有统一结构。
- 服务端保留完整日志。

缺点：

- 所有错误都变成同一句话。
- 缺少业务错误码。
- HTTP 状态可能仍是 200。
- 参数绑定和校验异常不一定都属于 RuntimeException 的理想处理范围。

## SystemConstants

public static final String IMAGE_UPLOAD_DIR =
        System.getProperty("user.home")
        + "/review-redis-backend/uploads/";
public static final String USER_NICK_NAME_PREFIX = "user_";

## 图片上传

入口：

```java
@PostMapping("blog")
public Result uploadImage(
        @RequestParam("file") MultipartFile image) {
    String originalFilename = image.getOriginalFilename();
    String fileName = createNewFileName(originalFilename);
    image.transferTo(new File(IMAGE_UPLOAD_DIR, fileName));
    return Result.ok(fileName);
}
```

### 文件名和目录散列

```java
String suffix = StrUtil.subAfter(originalFilename, ".", true);
String name = UUID.randomUUID().toString();
int hash = name.hashCode();
int d1 = hash & 0xF;
int d2 = (hash >> 4) & 0xF;
```

使用 UUID 防止同名覆盖，再用两个 0~15 的目录层级分散文件，避免一个目录堆积过多文件。

### 当前上传安全问题

- 没校验 MIME 类型和扩展名。
- 没限制文件大小和图片尺寸。
- `/upload/**` 被匿名放行。
- 删除接口是 GET，且只检查 `isDirectory()`。
- 没规范化并验证最终路径是否仍在上传根目录内，存在路径穿越风险。
- 本地磁盘不适合多实例共享。

生产系统应使用鉴权、白名单、随机服务端扩展名、大小限制、病毒/内容检查和对象存储。

## PasswordEncoder

```java
String salt = RandomUtil.randomString(20);
return salt + "@" + DigestUtils.md5DigestAsHex(
        (password + salt).getBytes(UTF_8)
);
```

它使用盐避免相同密码产生相同结果，但 MD5 计算太快，不适合现代密码存储。推荐 BCrypt、scrypt 或 Argon2。

当前验证码登录没有调用 PasswordEncoder，所以不能说项目已经实现安全密码登录。

## 正则工具

`RegexPatterns` 定义手机号、邮箱、密码、验证码格式；`RegexUtils` 把“空值或不匹配”统一视为 invalid。

当前只在登录链路使用手机号校验。正则规则也会随运营地区和号码段变化，不应视为永久真理。

## MyBatis-Plus 空壳模块

以下模块只有 Entity/Mapper/Service 基础结构：

- BlogComments。
- Follow。
- UserInfo 的写业务。
- SeckillVoucher 的自定义业务由 VoucherService 间接完成。

`ServiceImpl` 让它们具备通用 CRUD 能力，但没有 Controller 方法就不形成完整产品功能。

## WebExceptionAdvice 与 Result 的边界

业务可预期失败，例如库存不足，应该返回：

```java
Result.fail("库存不足")
```

非预期异常，例如数据库连接失败，才交给 Advice。不要用异常代替所有普通业务分支，否则日志会被可预期事件淹没。

## pom.xml 的构建配置

项目明确设置：

```xml
<java.version>21</java.version>
<maven.compiler.release>21</maven.compiler.release>
<lombok.version>1.18.38</lombok.version>
```

并把 Lombok 放入 `annotationProcessorPaths`，因为较新 JDK 对注解处理器自动发现更严格。

当前 README 仍写 JDK 17，文档和构建真值不一致。判断运行要求时以 `pom.xml` 和实际构建输出为准。

MySQL 驱动坐标 `mysql:mysql-connector-java` 在构建时提示已迁移到 `com.mysql:mysql-connector-j`，这是可维护性提醒，不影响当前构建成功。

## 面试讲法

> 项目配置通过环境变量覆盖 MySQL 和 Redis 地址密码，Spring MVC 用统一 Result 和 RestControllerAdvice 处理响应。MyBatis-Plus 配置分页拦截器，Redis 同时提供 StringRedisTemplate 和 RedissonClient。上传、密码工具和部分 CRUD 模块属于辅助能力，其中上传缺少生产级安全校验，PasswordEncoder 仍是盐加 MD5且未进入当前登录链路，我会明确把这些列为改进项。
