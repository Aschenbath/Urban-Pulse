# Urban-Pulse 后端重新学习手册

这套资料不是给项目“贴金”的宣传稿，而是一套从源码出发的复习教材。目标是让你重新建立完整的代码地图，知道每个模块为什么出现、先写什么、后写什么，以及面试时哪些能讲、哪些不能冒领。

## 先记住项目定位

> Urban-Pulse 是一个本地生活点评场景的 Java 后端练习项目。核心学习内容是 Spring Boot、MyBatis-Plus、MySQL 与 Redis，尤其是登录态、缓存和 Redis Stream 秒杀异步下单。Vue 2/nginx 页面是配套演示客户端，不作为独立完成的前端成果。

## 学到什么程度才算会

你不需要逐字背源码，但必须能够做到：

- 不看资料说清完整架构和个人贡献边界。
- 从 `POST /user/login` 讲到 Redis Hash token 和 ThreadLocal 清理。
- 从 `GET /shop/{id}` 讲到缓存穿透空值和 Cache Aside。
- 从 `POST /voucher-order/seckill/{id}` 讲到 Lua、Stream、事务、ACK 和 pending-list。
- 解释当前代码真正启用了什么，哪些只是注释中的备选方案。
- 主动指出 TTL 单位、固定消费者名、缺少数据库唯一约束和测试不足。
- 被问到前端时坦然说明它是配套演示客户端。

## 阅读标记

文档使用三种标记：

- **当前实现**：仓库现在真实运行的代码。
- **仓库备选**：代码存在，但当前调用路径未启用或被注释。
- **推荐改进**：为了生产可靠性建议增加，不能说成已经实现。
